Pokemon Terminal-Roguelike

Async fetch · Battle logic prototype · JSON heavy

A terminal roguelike prototype: random area + starter selection, live move + stat retrieval from PokéAPI, ASCII Pokémon display, and a turn‑based battle loop (damage, basic status, move selection). Core focus: learning async I/O, modular data shaping, and reducing startup latency from 193.5s (sequential) to about 20s (batched asyncio calls).

  • Async layer: Parallel move & type fetch (aiohttp) instead of blocking loops.
  • Data shaping: Normalizes raw JSON into move maps + level buckets for future leveling logic.
  • Battle engine start: Damage calc, type multiplier lookup per move, partial status effects (burn / sleep / confusion).
  • Refactor targets: Remove sync requests, seedable RNG, caching + retry, pure battle core.

Google-Drive-Automation

Drive API · Link normalization · Export mapping

Utility that ingests shared Google Drive file / folder URLs, resolves file IDs, queries the Drive API, determines the optimal downloadable form (native Google Docs exported to PDF/DOCX, others via direct download), and writes a clean manifest of stable download links locally (JSON / CSV). Focus: API auth flow, mimeType → export mapping, robust pattern parsing, and idempotent batch processing.

  • Normalized parsing: Regex + fallback logic to extract file IDs from multiple share URL shapes.
  • Export intelligence: Maps Google mimeTypes (Docs / Sheets / Slides) to preferred export formats.
  • Efficient batching: Minimizes API round‑trips (fields filtering + grouped metadata requests).
  • Reliable manifest: Deduplicates entries; persists structured JSON & optional CSV for downstream scripts.
